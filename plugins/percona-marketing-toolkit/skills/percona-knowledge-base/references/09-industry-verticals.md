## Financial services

**Why Percona is relevant:**
- Banks and financial institutions run large-scale MySQL, PostgreSQL, and MongoDB deployments
- Strict regulatory and compliance requirements (DORA in EU/UK, PCI-DSS, SOX, GDPR)
- High transaction volumes requiring performance and reliability
- Cost pressure to move off proprietary database licensing (Oracle, MongoDB Enterprise)
- Data sovereignty requirements

**Common database challenges:**
- Oracle-to-PostgreSQL migration (cost, lock-in, audit risk)
- MongoDB Enterprise licensing costs at scale
- Meeting DORA operational resilience requirements
- Multi-database environments spanning decades of technology choices
- 24x7 uptime requirements for payments and trading systems

**Regulatory/compliance context:**
- DORA (EU/UK financial institutions): operational resilience, ICT risk management
- PCI-DSS: payment card data security
- SOX: financial reporting controls
- GDPR: data protection and sovereignty
- Basel III/IV: operational risk frameworks

**Proof points:** BBVA, PayPal, Chime Bank, Merchant Warrior, Layer7, Fiserv

**Common competitor incumbents:** Oracle, MongoDB Enterprise, AWS RDS/Aurora, Azure Database

**Entry point products:** PSMDB (MongoDB Enterprise replacement), Percona Distribution for PostgreSQL (Oracle migration landing zone), Expert Support, PMM

## Technology and SaaS

**Why Percona is relevant:**
- Tech companies run complex, high-scale database infrastructure
- Developer-led adoption of open source databases
- Cost optimization at scale (cloud spend management)
- Multi-database environments (MySQL for transactional, MongoDB for document, PostgreSQL for analytics)
- Kubernetes-native infrastructure

**Common database challenges:**
- Scaling databases for rapid user growth
- Reducing cloud database costs (RDS/Aurora at scale gets expensive)
- Managing multi-database environments with small DBA teams
- Running databases on Kubernetes
- Database observability across services

**Proof points:** Cisco Systems, Slack, Vimeo, Duolingo, Appsuite, Alkemics, CouponCabin

**Common competitor incumbents:** AWS RDS/Aurora, Google Cloud SQL, MongoDB Atlas

**Entry point products:** PMM, Percona Operators, Expert Support, ExpertOps

## Retail and ecommerce

**Why Percona is relevant:**
- High-traffic ecommerce platforms run on MySQL and PostgreSQL
- Seasonal traffic spikes (Black Friday, holiday sales) require reliable scaling
- Cost pressure on database infrastructure
- Digital transformation driving new database requirements

**Common database challenges:**
- Handling traffic spikes without downtime
- MySQL performance optimization for high-volume transactions
- Migrating from proprietary to open source databases
- Database management while IT teams focus on digital transformation

**Proof points:** Toolstation, CouponCabin, Otto Office, Rent the Runway

**Common competitor incumbents:** Oracle, MySQL Enterprise, AWS RDS

**Entry point products:** Percona Server for MySQL, PXC (high availability), Expert Support, ExpertOps
