# Percona: Change Log

A dated record of every significant change to Percona's product, service, and branding landscape. Check this file to verify current product status and avoid referencing deprecated items.

---

## How to use this file

- Before writing any Percona copy, scan recent entries to catch anything that's changed
- Entries are listed most-recent-first
- Each entry states: what changed, and what to do in copy

---

## 2026

### January 2026: Percona Everest becomes OpenEverest

- **What changed:** Percona Everest transitioned to an independent open source project called OpenEverest, managed by Solanica (Percona wholly-owned subsidiary). Intended for CNCF donation.
- **What to do in copy:**
  - Never use "Percona Everest" in new copy
  - Refer to "OpenEverest" as an independent project
  - Percona is a contributor and provides enterprise support for OpenEverest
  - Existing Percona Everest customers see no immediate disruption
  - License remains Apache 2.0

### February 2026: PMM 3.6.0 released

- **What changed:** PMM 3.6.0 released with redesigned interface, native PMM navigation, High Availability Cluster (technical preview), simplified MongoDB dashboards, PostgreSQL 18 support on RHEL 10 and Debian 13
- **What to do in copy:** PMM version references should reflect 3.x series

### February 2026: Percona XtraBackup 8.4.0-5 community release

- **What changed:** Reduced backup lock time feature (previously Pro Builds only) now included in community release
- **What to do in copy:** This feature is no longer Pro Builds exclusive

## 2025

### December 2025: Percona Packages launched

- **What changed:** New suite of structured, fixed-scope consulting and support offerings. Initial packages: Quickstart (high-traffic event preparation) and AI Readiness.
- **What to do in copy:** Can reference Percona Packages as a new service offering

### October 2025: Peter Farkas appointed CEO

- **What changed:** Peter Farkas replaced Peter Zaitsev as CEO. Bennie Grant returned from Interim CEO to COO role.
- **What to do in copy:** Peter Farkas is CEO. Peter Zaitsev is Founder (no longer CEO).

### Mid-2025: 24x7 enterprise support for Valkey launched

- **What changed:** Percona expanded enterprise-grade support to cover Valkey
- **What to do in copy:** Can reference Percona support for Valkey. Percona does NOT develop Valkey software.

### Early 2025: Transparent Data Encryption (TDE) for PostgreSQL launched

- **What changed:** Percona introduced the first fully open source TDE implementation for PostgreSQL
- **What to do in copy:** Can reference as a key differentiator for Percona Distribution for PostgreSQL

### 2025: OpenID Connect (OIDC) support for PSMDB

- **What changed:** Percona added OIDC authentication support to Percona Server for MongoDB
- **What to do in copy:** Can reference as an enterprise feature of PSMDB

### 2025: Redis re-licensed to AGPL

- **What changed:** Redis Ltd re-licensed Redis to AGPL (from RSALv2/SSPLv1). Redis is now technically "open source" again under AGPL.
- **What to do in copy:** Redis can be called "open source" (AGPL is OSI-approved). Note it is "company-governed."

## 2021

### TokuDB deprecated

- **What changed:** TokuDB storage engine for MySQL deprecated
- **What to do in copy:** Do not reference TokuDB as a current product

---

## 2018

### MongoDB SSPL license change

- **What changed:** MongoDB, Inc. changed MongoDB's license from AGPL to SSPL (Server Side Public License). SSPL is not OSI-approved.
- **What to do in copy:** Percona Server for MongoDB (PSMDB) inherited SSPL license from upstream. Never call PSMDB "open source." Say "source available" or "freely available."

---

## Pre-2018

### 2015: Tokutek acquisition

- **What changed:** Percona acquired Tokutek, gaining TokuDB and TokuMX technology
- **What to do in copy:** Historical context only. TokuDB is deprecated.

### 2006: Percona founded

- **What changed:** Peter Zaitsev and Vadim Tkachenko founded Percona
- **What to do in copy:** "Founded in 2006" is the standard reference. 2026 is Percona's 20th year.
