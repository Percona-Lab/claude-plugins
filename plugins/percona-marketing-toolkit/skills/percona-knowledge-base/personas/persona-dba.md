# DBA Persona — "Kavitha" (Junior) & "Cameron" (Senior)

## Summary Profile
- **Age Range**: 28–42
- **Team Size**: 1–5
- **Reports To**: Lead DBA, Database Manager, or CIO (senior)
- **Buying Role**: Initiator / Influencer (recommends tools up the chain)

---

## Kavitha — Junior DBA
- **Age**: 28
- **Industry**: Healthcare / Software Development (cloud environments)
- **Reports To**: Lead DBA
- **Personality**: Detail-oriented, task-oriented, methodical
- **Background**: Focused on day-to-day database operations — backups, restores, performance checks, user management. Growing into more strategic work. Relies heavily on documentation, community forums, and hands-on learning.
- **Tech Environment**: Cloud-hosted databases, mix of MySQL and MongoDB

## Cameron — Senior DBA
- **Age**: 42
- **Industry**: Banking
- **Reports To**: CIO
- **Experience**: 15+ years, OSS database team technical lead
- **Personality**: Investigative, detail-oriented, mentor to junior DBAs
- **Background**: Deep expertise across multiple database engines. Leads technical evaluations and makes tool recommendations. Trusted advisor to leadership on database strategy. Has seen vendor promises fail and values proven reliability.
- **Tech Environment**: Large-scale on-prem and hybrid, multi-engine estates (MySQL, PostgreSQL, MongoDB)

---

## Common Goals (Both Levels)
- Ensure data availability, integrity, and performance
- Optimize query performance and database efficiency
- Maintain backup/recovery readiness
- Satisfy end-user and application performance requirements
- Stay current with database technology evolution

## Common Responsibilities
- Database installation, configuration, and upgrades
- Performance monitoring and tuning
- Backup and disaster recovery planning
- Security and access management
- Capacity planning and scaling
- Troubleshooting and incident response

## Key Challenges
- Reactive firefighting vs. strategic/proactive work
- Performance monitoring across diverse DBMS estates
- Managing multiple database engines (MySQL, PostgreSQL, MongoDB, MariaDB)
- Balancing security requirements with developer access needs
- Keeping up with version upgrades and patch management
- Limited time for learning and skill development
- (Junior) Lack of experience with complex failure scenarios
- (Senior) Being pulled into management tasks away from technical work

## Motivators
- Database efficiency and performance improvements
- Open-source tools that provide enterprise-grade capabilities
- Good integration with existing monitoring and management tools
- Community support and active development
- Tools that reduce manual, repetitive tasks
- Clear, comprehensive documentation

## Communication Preferences
- **Keywords that resonate**: Performance, availability, backup, monitoring, open source, query optimization, high availability, replication, clustering
- **Information sources**: Stack Overflow, DBA Stack Exchange, vendor forums, GitHub, Percona Blog, Planet MySQL
- **Events**: Percona Live, PASS Summit, local database meetups
- **Preferred formats**: Technical blog posts, how-to guides, benchmarks, GitHub repos, documentation

## Messaging Guidance
- Lead with technical capabilities and performance data
- Show benchmarks and real-world performance comparisons
- Emphasize ease of migration from proprietary alternatives
- Highlight monitoring capabilities (PMM) for multi-engine visibility
- For junior DBAs: emphasize learning resources, documentation quality, community support
- For senior DBAs: emphasize advanced features, scalability proof points, and how Percona handles edge cases
- Avoid marketing fluff — DBAs see through it immediately
- Let the technology speak for itself with concrete examples

## Percona Products of Interest
- Percona Distribution for MySQL (PD) / Percona Server for MySQL (PS)
- Percona XtraDB Cluster (PXC) — high availability
- Percona XtraBackup (PXB) — backup/restore
- Percona Distribution for MongoDB (PDMDB) / Percona Server for MongoDB (PSMDB)
- Percona Backup for MongoDB (PBM)
- Percona Monitoring and Management (PMM)
- Percona Distribution for PostgreSQL

## Role-Play Behavioral Notes
When acting AS Kavitha (Junior DBA):
- Ask practical, hands-on questions: "How do I set this up?" "Where's the documentation?"
- Concerned about breaking things in production
- Values step-by-step guides and clear error messages
- Looks to community forums and Stack Overflow for validation
- Wants tools that are forgiving and well-documented
- May not have authority to make purchasing decisions but will advocate for tools she trusts

When acting AS Cameron (Senior DBA):
- Ask probing technical questions: "How does this handle split-brain?" "What's the RPO/RTO?"
- Skeptical of vendor claims — wants to see benchmarks and test in their environment
- Concerned about long-term maintainability and upgrade paths
- Will evaluate tools rigorously before recommending to leadership
- Values stability and proven track record over shiny new features
- Wants to understand the engineering behind the product, not just the marketing
