# Sample Collection Guide

Tool-specific instructions for pulling writing samples during LEARN mode.
Use connected tools when available — pulling real samples beats asking
the user to paste.

---

## Gmail (for `email-internal`, `email-external`)

- Search sent mail: `in:sent` filtered by date range or recipient type
- For external style: `in:sent to:-(@yourcompany.com)`
- For internal style: `in:sent to:@yourcompany.com`
- Pull 8–12 sent email bodies; strip signatures before analysis
- Focus on emails the user composed (not forwards or one-word replies)

---

## Slack (for all `slack-*` contexts)

- Search sent messages by channel type (DM vs channel)
- For `slack-dm-tech`: messages in DMs with known engineering colleagues
- For `slack-dm-biz`: messages in DMs with non-technical colleagues
- For `slack-channel-team`: messages in team/squad channels
- For `slack-channel-company`: messages in #general, #announcements, or equivalent
- Look for substantive messages (not just "👍" or "thanks") — short messages
  are fine, but get ones with actual content
- Thread starters and replies may show different patterns — grab both

---

## Google Docs / Drive (for `notion-internal`, `doc-comment`, `blog-public`)

- Search Drive for docs the user authored
- For comments: look for documents with comment threads where the user is author
- Pull document bodies for writing style; pull comment text separately
- Comments are a distinct register from doc body — collect them as separate samples

---

## Google Calendar (for `calendar-invite`)

- Pull recent events the user created (not just accepted)
- Capture: event title, description/agenda field, any notes
- Focus on events where the user wrote the description, not auto-generated ones
- Need at least 5–8 events with user-written descriptions to build a useful profile

---

## Notion (for `notion-internal`, `doc-comment`)

- Pull pages authored by the user
- Inline comments are a separate sample set from page body text — collect both
- For page body: look for pages with substantial prose, not just tables or databases

---

## When no connectors are available

Ask the user to paste 5–10 samples directly. Coach them:

> "Paste 5–10 real messages you've sent in this context. Don't polish them —
> I need to see how you actually write, not your best version. Quick DMs,
> normal emails, regular posts. The more typical, the better."

---

## Multilingual sample collection

When collecting samples for a multilingual user:

- Filter by language when searching tools. For Gmail, add language-specific
  keywords or recipient domains to narrow results. For Slack, filter by
  channels known to use a specific language.
- Keep samples in a single language per extraction pass. Don't mix English
  and Portuguese samples into one analysis — the patterns will blur.
- If the user code-switches within messages (common in multilingual teams),
  note that as a style dimension rather than filtering it out.
